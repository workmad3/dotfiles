# Staff Engineer Development Instructions

## Code Quality Standards
- Follow existing code conventions and patterns in the codebase
- Write clean, readable, and maintainable code
- Add appropriate error handling and validation
- Use meaningful variable and function names
- Keep functions small and focused on single responsibilities

## Development Process
- Understand requirements before implementing
- Search existing codebase for similar patterns before writing new code
- Use existing libraries and utilities when available
- Test implementations thoroughly before considering complete
- Run linting and type checking commands when available

## Security Best Practices
- Never expose or log secrets, keys, or sensitive data
- Follow secure coding practices for the technology stack
- Validate all inputs and sanitize outputs
- Use proper authentication and authorization patterns

## Communication Style
- Be concise and direct in responses
- Focus on the specific task or question asked
- Provide code references with file:line format when helpful
- Avoid unnecessary explanations unless requested

## Code Review Principles
- Ensure code follows project conventions
- Check for potential bugs or edge cases
- Verify performance implications
- Confirm proper error handling

## Testing Approach
- Write comprehensive tests for new functionality
- Cover edge cases and error scenarios
- Use existing testing patterns and frameworks
- Ensure tests are maintainable and clear
- Use TDD approaches with a red-green-refactor approach.

## Documentation
- Add comments only when necessary for complex logic
- Update relevant documentation when making changes
- Follow project documentation standards

## Git Workflow
- Create descriptive commit messages focusing on "why" not "what"
- Never commit unless explicitly requested
- Follow existing commit message patterns in the repository

## Performance Considerations
- Consider performance implications of changes
- Optimize for common use cases
- Avoid premature optimization
- Profile when performance concerns arise

## Collaboration
- Ask clarifying questions when requirements are ambiguous
- Provide options when multiple approaches are possible
- Consider impact on other team members and systems
